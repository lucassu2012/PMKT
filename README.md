# PMKT · 存量经营分省作战 · 在线作战协同原型

面向中国移动 31 省存量用户精细化经营（离网、降档、降套、升套）的内部作战协同原型。主入口为 `index.html`，双击即可离线运行；请保持 `province-data.js` 与入口文件位于同一目录。

> 本项目是交互与规则原型，不是生产权限系统。建设蓝图中的华为云低代码、系统 DB、SSO/RBAC、工作流、审计和对象存储仍需在生产实施阶段落地。

在线访问：启用 GitHub Pages 后见 https://lucassu2012.github.io/PMKT/

## 当前数据基线

- 数据来源：`省份数据.xlsx`
- 数据批次：2026-07-10
- 覆盖范围：31/31 省；两张工作表按省份关联，无重复或缺省省份
- 数据内容：客户 TOP1 意愿、数据接入与治理、模型能力、经营挑战、网络体验权益、7 类现网平台、波次及路径、9 项经营 KPI
- 判级纪律：Excel 未提供完整的四维 L1-L5 证据链，因此只刷新原始业务数据、波次、KPI 与平台格局，不静默覆盖正式判级
- 格局映射：大数据和 IOP 均为华为=强格局；其一为华为=部分格局；其余=友商格局
- 波次纪律：源表只提供 1/2/3，系统仅展示原始波次，不推导完成率

系统显式保留并提示源数据问题，包括占位表头“XXXX合约率”、新疆跨表空值/0 冲突、部分超过 100% 的指标、同比单位缺失和供应商简称不一致。

## 功能

- **态势总览**：全国/本省关键指标、优先作战队列、四维均值、波次分布和数据质量待办。
- **省份作战**：地图、表格和双轴矩阵联动；按梯队、加权分、置信度或平台格局查看省份。
- **省份档案**：统一展示正式判级、Excel 经营 KPI、波次、客户信号、数据/模型基础、平台格局、质量问题和开方建议。
- **判级向导**：每维度决策树，支持“是 / 否 / 不确定”；不确定项自动转为情报任务。
- **情报闭环**：待派 → 回填中 → 待审核 → 已入库；入库后重算梯队并将旧剧本标记为需复核。
- **作战策略**：四维短板 + 梯队 + 内部格局生成方案组合、依赖链、模式 A/B 和规则剧本草稿。
- **数据治理**：展示 Excel 批次、字段映射、31 省明细、数据质量和变更日志。
- **角色视图**：机关、代表处、领导三种原型视图；代表处仅看本省且隐藏平台格局，领导只读。
- **安全导出**：内部快照包含波次/KPI/格局；对外版从白名单字段独立生成，不读取内部格局和省份判断字段。

## 评估方法

四维加权：数据准备度 30%、场景化模型能力 20%、营销闭环能力 30%、体验经营 20%。

`加权分 = 数据×0.3 + 模型×0.2 + 营销×0.3 + 体验×0.2`

- T1：≥4.0
- T2：3.0–3.9
- T3：2.0–2.9
- T4：<2.0
- 数据准备度 <L3 时，梯队强制封顶 T3

## 本地状态与版本迁移

原型使用浏览器 `localStorage` 保存判级、任务、日志和剧本草稿。状态包含 `schemaVersion` 与 `dataVersion`：数据批次更新时，系统刷新 Excel 基线字段，同时保留本地人工判级和任务记录；旧版 `zlyj_battle_v3` 会迁移到 v4。

页面中的“重置数据基线”会清空本地人工改动并恢复 2026-07-10 Excel 基线，仅机关视图可触发。

## 技术与边界

- 原生 HTML/CSS/JavaScript，无服务端依赖
- `province-data.js` 是由 `省份数据.xlsx` 结构化生成的只读数据资产
- 中国地图为 DataV GeoJSON 简化投影后的内联 SVG，仅作内部原型示意；生产发布前必须完成地图合规核验并补充审图号
- 角色切换只用于演示视图裁剪，不能替代生产环境的 SSO、RBAC、字段级权限与服务端审计
