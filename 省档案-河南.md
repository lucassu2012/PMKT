# 省档案 · 河南

## 1. 一句话画像

「数据/AI 使能存量经营」的中国区提值标杆省：商机转化率 5%→6.8%（+35%）、25 年高价值用户净增 100 万+、MBB 收入年增 3%，且高铁包体验变现（VIP 短视频高清 +26%）与防降档促升套均已落地——数据长板扎实、体验商用有据；短板在存量经营 LUM 大模型与实时营销通道的证据尚缺。属**数据与体验双长板的 T2 发展梯队省**，中部规模复制样板候选。

## 4. 华为格局轴判定（仅内部版）

## 6. 开方建议（T2 补短板放大 → 冲 T1）

主逻辑：数据（L4）与体验（L4）双长板已成，距 T1 仅差模型或营销一档——**用 ② 把已上市的防降档促升套从规则模型升级为 LUM 泛化，用 ③ 补实时营销，即可升 T1 样板深化**（开方框架见 [[02_战略规划/存量经营分省作战/开方逻辑-四大方案映射]]）。

| 优先级 | 方案 | 打法 |
| --- | --- | --- |
| 主打冲档 | ② LUM 大模型泛化预测（降档/升套/离网） | 数据 L4 已过门槛，可直接上模型：河南防降档促升套已按套餐饱和度分层运营，用 LUM 替代规则阈值、提升防降档查准查全，把「向体验要增量」的提值故事做成个体级预测。依赖链约束：确认 ≥3 个月 O+B 融合数据就绪；算力可承接 Token Plan 经营底座 |
| 主打做深 | ④ 无线智能板+NWDAF 网络权益运营 | 高铁包 +26%、京广线 ROI 3.87 已是全体系商用锚点，横向复制场景权益：三件套话术（集团《客户分层分级网络保障服务业务规范》合规背书 + 已商用套餐模板库[天津地铁包/北京工体包/广东场馆包] + 商用数据[河南高铁包 +26%、北京工体 VIP 速率 +4.4 倍且普通用户无损]）。价值主张=高价值用户保有+权益增收双轮（正文见 [[03_产品方案竞争力/无线智能化运营/无线智能化运营]]，不复制） |
| 补短板 | ③ 精细化营销 | 格局待核实：默认按**模式 B**（LUM 单卖+API 输出被集成到客户自有大数据/营销平台，华为不碰执行层）起步；若核实为强格局则切换**模式 A**（NWDAF→FI Kafka→IOP 端到端实时营销旗舰）；实时营销场景以 ④NWDAF 就绪为前提 |
| 格局反制 | 竞争防御 | 智算运维已丢标卓望——在大数据/IOP 与 ADO/Agentic BSS 无风险样板通路上抢先入局、把华为能力写入需求，避免运维域格局失守外溢到经营域 |

## 7. 情报缺口（代表处校准任务）

1. 河南移动大数据平台与 IOP 在网厂商是谁？卓望在河南的地盘边界（仅智算运维还是含经分/营销）？（定格局轴与方案③模式选择）
2. 防降档促升套是规则模型还是已用预测模型？是否有 LUM/Smart Decision 河南试点？（校准模型维 L3、定 ② 切入点）
3. 「营销精准触达」是否已打通事件驱动实时营销（NWDAF/事件→Kafka→IOP）？（定营销闭环是否达 L4、冲 T1 的关键一档）
4. 防降档促升套的脱敏收益（源 PPT「X 千万」）实际数字，以及离网率/降档率基线？（回填 KPI 佐证）
