# 在线作战系统 · 建设蓝图（存量经营分省作战）

## 一句话定位

把本目录 12 张作战卡从"静态文档"升级为**华为内部在线作战协同平台**（分期建）：**机关主用、代表处轻回填、领导只读**；系统 DB 成为 31 省判级数据的**运营真相源**，Obsidian 卡片降为系统**单向导出的只读快照**。首期打通"情报校准闭环"，用**低代码在华为云**落地。

## 2. 数据模型（低代码里就是几张关联表）

| 表 | 关键字段 | 说明 |
|----|----------|------|
| **省份** | 省名、数据L、模型L、营销L、体验L、加权分、梯队、格局轴、总置信度 | 31 行；**加权分/梯队为公式列**：`加权分=数据×0.3+模型×0.2+营销×0.3+体验×0.2`，梯队按分数段并叠加"数据<L3 封顶 T3"逻辑，均自动算，不手填 |
| **判级明细** | 省、维度、L 级、置信度、判级依据、证据链、速查问答留痕 | 省×4 维=124 行；每格必附置信度；证据链指向 KB 卡双链或上传情报 |
| **情报任务** | 省、维度、状态（待派/回填中/待审核/已入库）、承办代表处、提报内容、审核结论 | **killer 闭环的载体**；低置信格自动生成 |
| **规则配置** | 权重、梯队分数段、门槛规则、判级速查题库、开方映射 | 从 [[02_战略规划/存量经营分省作战/评估标准-四维L1-L5]]、[[02_战略规划/存量经营分省作战/开方逻辑-四大方案映射]] 同步；**可配置不硬编码**，规则定稿不改系统 |
| **变更日志** | 时间、操作人、省、维度、前值→后值、依据 | 可追溯 + 可回滚 |

## 3. 关键流程

### 3.1 判级/打分（机关）
速查向导（每维度 3 问二选一）→ 自动推 L 级 → 系统实时算加权分/定梯队/灰掉被门槛封顶的档 → 必填置信度 + 挂证据 → 保存。专家已知的省可直接下拉改级。

### 3.2 情报校准闭环（机关 × 代表处）
机关在地图上标出低置信格 → 系统生成情报任务派给对应代表处 → 代表处在"我的情报任务"回填证据+建议判级 → 进机关审核队列 → 机关通过 → 更新 L 级/升置信度/**自动重算分档与梯队** → 地图与打单优先级刷新。全程留痕。

### 3.3 markdown 快照导出（系统 → KB）
机关一键触发 → 系统按当前 DB 数据生成 [[02_战略规划/存量经营分省作战/分档地图-31省双轴矩阵]] 与各省档案的 markdown 快照 → 覆盖 KB 中的只读快照。**数据改在系统、规则改在 KB**，不反向手改省份数据。

## 4. 分期路线

| 期 | 交付 | 内容 |
|----|------|------|
| **一期（MVP）** | 地图 + 判级向导 + 情报闭环 | 31 省交互地图（读+筛选+钻取省档案）、判级速查向导自动打分、情报提报-审核-重算闭环、三内部角色权限、12 卡初始化导入、markdown 单向导出 |
| **二期** | 决策增强 | 开方规则引擎（结构化作战建议卡）、双轴打单优先级矩阵可视化、对外裁剪版导出（PDF/PPT，机械摘除格局轴+排名）、领导驾驶舱 |
| **三期** | 智能增强 | AI 生成作战剧本文案（人工审核后用）、与源系统数据对接（选配） |

---

## 5. 落地纪律（执行时守住）

1. **规则可配置**：权重/梯队分数段/门槛/速查题库/开方映射一律做成配置项，规则仍处"待复核"（[[02_战略规划/存量经营分省作战/评估标准-四维L1-L5]] 第 8 节挂了 LUM 口径矛盾与梯子待校准），定稿后改配置不改系统。
2. **单向导出不可逆**：上线后省份数据只改在系统，KB 快照会被下次导出覆盖，须向团队讲清。
3. **格局轴脱敏到对外链路**：对外只走生成的裁剪版导出，格局轴/省份排名永不进对外视图；代表处角色默认不见格局。
4. **初始种子=12 卡**：8 深判省为高/中置信，23 省为低置信初判，导入后即成情报任务清单，交代表处校准。
